# 更新日志

本文件记录 proj-lysis 的所有重要变更。

---

## [2.0.0] — 2026-06-07

### 🎉 新增

- **完整 TypeScript 重写** — 基于 magic-analyzer 重构，零外部运行时依赖
- **双排除配置** — `preset`（预设排除，含 25 项前端常见目录/文件）+ `exclude`（自定义追加）
- **体积统计** — 新增 `bytes`、`sizePercentage` 字段，统计同类文件总体积及占比
- **中英双语 Markdown 报告** — 4 个报告文件：
  - `report-zh.md` — 代码统计（中文表头）
  - `report-en.md` — Code statistics（English headers）
  - `size-zh.md` — 体积明细（中文，含单个文件路径和体积）
  - `size-en.md` — Size details（English，per-file paths and sizes）
- **三维护排序** — `sortField`: `files` | `codeLines` | `codePercentage`，均支持升序/降序
- **注释行统计** — 支持 50+ 编程语言的注释语法检测（单行 + 块注释）
- **`.gitignore` 自动读取** — 自动收集分析目录及祖先目录的 `.gitignore` 规则
- **CLI 命令行工具** — `proj-lysis` 命令，支持 `--dir`、`--preset`、`--exclude`、`--sort-field` 等
- **`.d.ts` 类型声明** — 构建自动生成类型文件，支持 TypeScript 消费者
- **rolldown 打包** — ESM + CJS 双产物

### 🔧 改进

- **无全局状态** — 每次 `analyze()` 调用独立计算，无副作用
- **扩展名处理优化** — 正确处理点文件（`.env` → `env`，`.gitignore` → `gitignore`）
- **行数统计算法** — 修复末尾换行符误计和空文件统计问题
- **二进制排除增强** — 从 7 种图片扩展到 40+ 种（含字体、音视频、压缩包）

### 🔒 安全修复

- **隐私保护** — `size-*.md` 中的文件路径使用相对路径，不暴露完整文件系统路径
- **DoS 防护** — `safeReadFile()` 添加 10 MB 文件读取上限
- **递归深度限制** — `collectFiles()` 最大深度 32 层，防止符号链接循环
- **输出路径验证** — `outputPath` 禁止写入分析目录之外
- **Markdown 注入防护** — `sanitizeMdTable()` 转义 `|` 等特殊字符
